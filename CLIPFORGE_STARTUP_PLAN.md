## Free Clip Search Notes

Best free workaround:
- Use only free sources inside the app.

Free clip sources:
- Pixabay API
- Wikimedia Commons
- Internet Archive
- Pexels API
- Wikipedia

Brave workaround without paying:
- Brave gives monthly credits and can be used only for better search phrases.

Best setup for no money:
- Use Brave only for better search phrases.
- Use Pixabay, Pexels, Wikimedia, and Internet Archive for the actual clips.
- Limit Brave to 1 search per video idea.
- Cache the results on your phone.
- Do not re-search the same subject twice.

Even better free version:
- Skip Brave completely.
- Use Wikipedia to get the fact.
- Use the subject plus scene words to search:
- Wikimedia Commons
- Internet Archive
- Pixabay
- Pexels

That keeps the app free.

## Why The App Showed Random Black-And-White Clips

The real problem:
- It searched Internet Archive too broadly.
- Internet Archive has tons of old black-and-white videos.
- Broad words like `speech`, `crowd`, `history`, `trial`, or `people` pulled random archive footage.
- It labeled clips as "exact topic" too easily.
- It fell back to related B-roll too fast.
- It searched for "legal clip available," not "this exact fact."

The fix should be strict:
- Exact subject results only show if the clip title, tags, or source page includes the subject.
- If no exact clips exist, it should say "No exact legal clip found."
- Related B-roll should be separate and clearly labeled.
- Internet Archive should be off by default unless the user chooses "archive footage."
- Black-and-white clips should only show for history or archive topics.
- Stock B-roll should come from Pixabay or Pexels, not random archive films.

Better app structure:
- Exact Clips
- Related B-roll
- No Match

That is the real issue.
- The app needed stricter filtering, not more random sources.
