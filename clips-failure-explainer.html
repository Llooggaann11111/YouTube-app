FactPulse source failure guide

What your log means

YouTube: 0 means no YouTube Data API key was entered, the key is wrong, quota is used up, or no embeddable matches were returned. Wikimedia: 0 or 2 means it searched correctly, but only found a few Creative Commons videos that matched the exact subject. Internet Archive: 8 means this source worked. Those are the usable clips the app found. Library of Congress: searched, then failed means the browser/proxy allowed some requests and blocked others. This is not a search logic problem. National Archives: failed or blocked means the API is not reliable from a static GitHub Pages browser app. It needs a backend proxy. DVIDS: failed or blocked means the API is not reliable from the browser. It needs a backend proxy. Pixabay, Pexels, Coverr: 0 means no key was entered, stock was off, or the API returned no stock matches.

Real fix

The search page is browser-only. Some government and archive APIs block browser requests. The reliable fix is a small backend proxy. It would search National Archives, DVIDS, Library of Congress, YouTube, Wikimedia, Internet Archive, Pixabay, Pexels, and Coverr from a server, then send clean results back to the app.

Without that backend, the app can only use sources that allow browser requests.

Use now

For celebrity clips, add a YouTube Data API key. For renderable background footage, add Pixabay and Pexels keys.
